PulsePoll uses a privacy by design approach. Vote choices are stored separately from voter identities. IP addresses are hashed and encrypted before storage. No one can link a vote to a real person. IP addresses are first hashed using SHA-256 — a one-way function that cannot be reversed. That hash is then combined with the poll ID and AES-256 encrypted before being stored in the database.

The vote choice itself is stored in a completely separate table with no direct link to the voter's identity. Even if someone accessed the database, they could not connect a vote to a person without the encryption key — which is never stored in the DB.

No. PulsePoll is built to be completely anonymous. As a poll creator you can see how many people voted and what they voted for — but never who. This is guaranteed by our architecture, not just policy. The database has two separate tables — one for votes (what was chosen) and one for vote identity (encrypted who voted). These tables are deliberately separated so that knowing one tells you nothing about the other.

Even Pulsepoll admins cannot reverse this process without the encryption key. Your voters' privacy is protected at the database level itself.

Every poll gets a unique shareable link. After creating a poll, go to your dashboard, click View on the poll card, and copy the share link from the poll detail page. Send it via WhatsApp, email, or any platform!

When a poll reaches its end time, voting closes automatically. The vote page shows a "Voting has ended" message and redirects to final results. The expiry is enforced at two levels for security:

1. The vote page checks the end time and shows the expired state — no voting form is shown.

2. Even if someone tries to submit a vote directly via the API, the server checks the end time and returns a 400 error. This means the expiry cannot be bypassed by technical users trying to manipulate the UI.

Results remain visible forever after expiry so anyone with the link can always view the final outcome.
